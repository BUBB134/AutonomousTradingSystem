# Runbooks

This directory will contain reviewed procedures for deterministic operations, incident response,
recovery, reconciliation, and isolated paper-simulator controls.

Runbooks must:

- state prerequisites and the permitted operating mode;
- identify commands and expected evidence;
- fail closed when state is ambiguous;
- include recovery and escalation steps; and
- never contain credentials or instructions for live trading.

No operational runbook exists yet because the current repository contains development
infrastructure only.
